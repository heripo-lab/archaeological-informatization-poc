# OpenAI API 키 발급 가이드

중요: 운영체제 지원 안내
- 본 PoC는 macOS와 Linux에서의 실행만을 공식적으로 지원합니다.
- Windows 네이티브 환경(Windows PowerShell/명령 프롬프트에서 직접 실행)은 지원하지 않습니다. Windows 사용자는 WSL(Windows Subsystem for Linux)에서 Ubuntu 등 Linux 배포판을 설치한 뒤, WSL 터미널에서 프로젝트를 실행해 주세요.
  - WSL이란? Windows에서 가벼운 Linux 환경을 구동할 수 있게 하는 기능입니다. 별도 가상머신 UI 없이 Windows와 나란히 Linux를 사용할 수 있습니다.
  - WSL 설치 가이드: https://learn.microsoft.com/ko-kr/windows/wsl/install
  - WSL 소개: https://learn.microsoft.com/ko-kr/windows/wsl/about
- 왜 Windows 네이티브를 지원하지 않나요?
  - 본 저장소는 개인 논문 연구용으로 macOS 기반 환경에 맞추어 만들어졌습니다.
  - 오픈소스 공개는 연구 투명성 공유가 목적이며, 전문 개발지식이 없는 사용자에게 실행을 권하지 않습니다. 공개 준비 과정에서도 추가 호환성 확보(특히 Windows 네이티브)는 진행하지 않았습니다.
  - Docker 제공도 고려했으나, Windows 사용자에게는 오히려 진입 장벽이 될 수 있어 제공하지 않습니다.

---

먼저 다음 공식 페이지를 참고해 주세요(한국어):
- https://openai.com/ko-KR/index/openai-api/

이 문서는 위 페이지를 먼저 읽은 뒤, 실제 키 발급과 프로젝트 설정 방법을 간단히 정리한 보조 가이드입니다.
중요: API 키 관리와 사용량/비용 관리는 전적으로 실행자 본인의 책임이며, 저장소 소유자는 이에 대해 어떠한 책임도 지지 않습니다.

1) OpenAI 계정 생성/로그인
- OpenAI 계정이 없다면 가입을 진행합니다.
- 기업/기관 계정의 정책을 준수하세요(과금, 데이터 보안 등).

2) 결제 정보 확인(중요)
- 유료 API 사용에는 결제 수단 등록이 필요합니다. 과금 한도와 알림을 설정해 예기치 않은 비용 발생을 방지하세요.
- 비용/키 관리는 전적으로 실행자 본인의 책임이며, 본 저장소/작성자는 과금·운영상의 손해에 대해 책임지지 않습니다.

3) API 키 생성
- 일반적으로 OpenAI 플랫폼 콘솔에서 API 키를 생성합니다.
  - 예시 경로: https://platform.openai.com/api-keys (UI/주소는 시기별로 변경될 수 있음)
- Create new secret key 버튼을 눌러 새 키를 생성합니다.
- 발급 직후에만 전체 키 문자열을 볼 수 있으니, 즉시 안전한 비밀 저장소(예: 1Password, Bitwarden 등)에 보관하세요.

4) 프로젝트에 키 설정(.env.local)
- 이 저장소는 루트에 `.env.local` 파일을 사용합니다(예시 파일: `.env.local.example`).
- `.env.local` 파일에 다음과 같이 입력하세요:

```
OPENAI_API_KEY=sk-************************
```

- 주의: `.env.local`은 `.gitignore`에 포함되어 있으므로 키가 커밋되지 않습니다. 키는 절대 공개 저장소에 올리거나 공유하지 마세요.
- 또한 API 키 관리와 사용량/비용 관리는 실행자 본인의 책임입니다. 본 저장소/작성자는 이에 대해 책임지지 않습니다.

5) 보안/운영 팁
- 키 분실/유출이 의심되면 즉시 콘솔에서 키를 폐기(Rotate/Revoke)하고 새 키를 발급받으세요.
- 최소 권한 원칙: 필요 시 프로젝트별로 서로 다른 키를 발급해 용도를 분리하고, 사용 한도를 설정하세요.
- 비용 관리: 사용량 대시보드/알림을 활용해 과금 추이를 모니터링하세요.

6) 실행 전 확인
- `.env.local` 작성 후 앱을 빌드/실행하기 전에 터미널 세션을 새로 열어 환경 변수가 반영되었는지 확인하세요.
- 본 PoC는 LLM 호출량이 많을 수 있으므로, 로컬 테스트 범위에서만 제한적으로 실행하시기 바랍니다.
